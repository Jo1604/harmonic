Review: Types of Control Flow Structures in C

  • For Loops:
    1. Initialize: Executes once at the start. Usually a loop control variable
    2. Condition: Executes at the start of each iteration. If the condition is false, the loop will exit.
    3. Increment: Executes at the end of each iteration. Usually updates the loop control variable.
  • If /Else statements:
    1. Condition: Expression that determines if the statements in brackets will execute.
    2. Else statements: Executed if the condition is not met.
  • While Loops:
    1. Condition: Expression that determines if the statements in brackets will execute.

How to Flatten C Control Flow Structures

  • Flattening code:
    1. Helps get code into format similar to ARM for easy translating
    2. Includes adding labels and goto instructions
    3. Conditions are inverted

Translate If Statements to ARM with Example

  • Parts of Translation:
    1. Initialize: Do in .bss sections because variable not set equal to a value. Load the variable base address into register x0
    2. Condition: Easiest to use cmp instruction as a shortcut. The PSTATE Register will be set for conditional branch
    3. Goto: Jump to “else” or “endif” label with branch instructions (conditionally and unconditionally, respectively)

Translate For Loops to ARM with Example

  • Parts of Translation:

    2. Condition: Easiest to use cmp instruction as a shortcut. The PSTATE Register will be set for the conditional branch instruction

Signed vs. Unsigned Numbers

  • Basic Load, Store, and Address:
    1. Load (ldr): Derefrences source pointer register using [] and loads contents of source to destination register
    2. Store (str): Stores data from destination register to derefrenced source
  • Variations:
    1. Many variations exist to these formats to make refrenceing offsets easier
    2. Very helpful for data structures like arrays (see practice at the end of the lesson)

Sections and Data Types

  • Sections of Code:
    1. .data : For variables with known value (Read/Write)
    2. .rodata : For data you will only read from like strings.
    3. .bss : For vaiables without known value (set to zero)
    4. .text : For your code (Read only)
  • .Global label:
    1. label variables/data and sections that need to be seen from outside the file
  • Data types:
    1. char = .byte (1 byte)
    2. short = .short (2 bytes)
    3. int = .word (4 bytes)
    4. long = .quad (8 bytes)
