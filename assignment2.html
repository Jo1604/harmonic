High Level vs Low Level Languages

  • High level languages (e.g. Java, C, Python) are:
    1. Portable: Can run on different machines
    2. Complex: Highly abstract commands and lines
    3. Human Readable: Easy to read, usually in plain English
  • Low level languages like Assembly Languages (e.g. ARM, MIPS, x86) are:
    1. Not Portable: Can't run on different machines
    2. Simpler: Low abstractions mean more simple tasks per instruction
    3. Human Readable-ish: Can be with some practice

von Neumann Architecture

  • Control Unit:
    1. Holds current instruction
    2. Moves data between RAM, registers, and ALU
  • RAM:
    1. Large storage, Slow access
    2. Instructions and data stored the same way here
  • Registers:
    1. Small Storage, Fast access
    2. Data manipulation happens here
  • ALU:
    1. Computations happen here

Registers

  • In General:
    1. X0-X30 for 64 bit(or W0-W30 for 32 bit)
    2. X0-X7 as General Purpose (more on conventions later)
    3. Use to maninpulate small amounts of data quickly, pass parameters, and return values/addresses
  • Special Registers:
    1. Program Counter (PC): hold next instruction in program (not always sequential for branches)
    2. Stack Pointer (SP): points to top of stack frame (lowest address)
    3. Frame Pointer (FP = X29): points to top of main stack frame (lowest address)
    4. Link Register (LR = X30): Holds return address
    5. XZR: Register full of zeros. No data saved here

PSTATE Register

  • How it works:
    1. When doing arithmetic or logical instructions, flags may be set
    2. Flags: n (Negative), z (Zero), c (Carry), v (Overflow)
  • Instructions that use it:
    1. Signed Arithmetic: Might overflow or get negative numbers
    2. Compare Instruction: cmp sets flags for next instruction
    3. Conditional Instruction: Branch if equal (beq), Brach if greater than (bge)...

Learn more

Arithmetic Instructions

    Format 1: name dest , src1,src2

    Format 2: name dest , src1,imm

  • Format Key Words:
    1. dest: where will the calculation be saved
    2. src1 or src2: what data is used for the calculation
    3. imm: immediate value or constant to use in calculation
    4. dest and src can be the same register!
  • Common Instructions:
    1. Arithmetic: add, sub, mul, sdiv, ars
    2. Logic: and, oor, eor,

Learn more

Load and Store

  • Basic Load, Store, and Address:
    1. Load (ldr): Derefrences source pointer register using [] and loads contents of source to destination register
    2. Store (str): Stores data from destination register to derefrenced source
  • Variations:
    1. Many variations exist to these formats to make refrenceing offsets easier
    2. Very helpful for data structures like arrays (see practice at the end of the lesson)

Learn more

Sections and Data Types

  • Sections of Code:
    1. .data : For variables with known value (Read/Write)
    2. .rodata : For data you will only read from like strings.
    3. .bss : For vaiables without known value (set to zero)
    4. .text : For your code (Read only)
  • .Global label:
    1. label variables/data and sections that need to be seen from outside the file
  • Data types:
    1. char = .byte (1 byte)
    2. short = .short (2 bytes)
    3. int = .word (4 bytes)
    4. long = .quad (8 bytes)
